## 1. Locked decisions (do not revisit without explicit human instruction)

- **Product:** Campus ride-hailing for University of Ghana, Legon. Web v1 paused; this is a clean rebuild. Working name **"CampusRide"**, defined once as `APP_NAME` in [apps/server/src/config.ts](apps/server/src/config.ts).
- **Stack:** TypeScript everywhere, strict mode, no `any`. npm-workspaces monorepo: `packages/shared`, `packages/mobile-shared`, `apps/server`, `apps/rider`, `apps/driver`. Backend: Fastify (`apps/server/src/index.ts`). DB: PostgreSQL on Neon via Prisma (`DATABASE_URL` pooled + `DIRECT_URL` direct, see `apps/server/prisma/schema.prisma`). Mobile: Expo + Expo Router (SDK 51, deps via `expo install`, no `legacy-peer-deps`) â€” two apps, `apps/rider` (rider-facing) and `apps/driver` (minimal placeholder shell for Phase 6), both sharing design system + auth from `packages/mobile-shared`. Real-time: Socket.io, attached to the Fastify HTTP server in `apps/server/src/index.ts` (currently only logs connect/disconnect â€” no app events emitted yet). Payments: Paystack/Moolre planned for Phase 4 â€” `PaymentService` interface + `DummyPaymentService` exist now (`apps/server/src/services/payment/`). Maps: Mapbox display + Socket.io driver dot + dummy ETA planned, behind `RouteService`/`DummyRouteService` (`apps/server/src/services/route/`). OTP: mNotify planned â€” `OtpService`/`DummyOtpService` exist now (`apps/server/src/services/otp/`). Photos: Cloudinary (not yet integrated).
- **Money:** integer pesewas everywhere (GHS 1 = 100 pesewas), never floats. Defined in [packages/shared/src/pricing/pricing.ts](packages/shared/src/pricing/pricing.ts). Invariant `commission + driverShare === totalFare` holds exactly (`splitFare`, `COMMISSION_PCT: 15` / 85% to driver).
- **Ride model (TWO types)** â€” `RideType` enum (`LONE` | `SHARED`) in schema:
  - **LONE**: flat `LONE_FARE: 1500`, exactly 1 rider (`validateLoneOccupancy`), immediate.
  - **SHARED**: dynamic by final occupancy at departure, max `MAX_SHARED_OCCUPANCY: 4`. Per-rider rate via `SHARED_FARES = { 1: 1000, 2: 700, 3: 600, 4: 500 }` (`getSharedFarePerRider`); total per car = occupancy Ă— rate, i.e. **1000 / 1400 / 1800 / 2000** as occupancy goes 1â†’4 (`getSharedTotals`). Commission 15% platform / 85% driver on that total. Price locks per-passenger on join via `recomputeLockedFares` ([apps/server/src/services/ride/lockedFare.ts](apps/server/src/services/ride/lockedFare.ts)) â€” **downward-only** (a join can lower everyone's locked fare as occupancy grows, but a later cancellation never raises it back). Driver absorbs empty-seat risk; platform never subsidizes. Charged at departure. Riders can cancel any time before departure.
- **Dispatch:** broadcast to eligible drivers (online, approved, in pickup zone or an adjacent zone, no active ride) via `getEligibleDrivers` ([apps/server/src/services/ride/dispatch.ts](apps/server/src/services/ride/dispatch.ts)); first to claim wins via `claimRide`'s atomic conditional `updateMany` (`WHERE status='REQUESTED' AND driverId IS NULL`). 90s no-claim â†’ `AWAITING_RIDER_DECISION` â†’ rider picks `KEEP_WAITING` / `SWITCH_TO_LONE` (only if exactly 1 active passenger) / `CANCEL` via `riderDecision` ([apps/server/src/services/ride/riderDecision.ts](apps/server/src/services/ride/riderDecision.ts)); a further 90s grace with no decision â†’ `CANCELLED` / `NO_DRIVERS_AVAILABLE`. **Both 90s timeouts are owned entirely by `processTimeouts`** (`shouldTimeout`, `shouldExpireDecision` in [apps/server/src/services/ride/timeouts.ts](apps/server/src/services/ride/timeouts.ts)), polled every 15s from `bootstrap()` in `apps/server/src/index.ts`. **There is intentionally NO `broadcast.ts` and NO `TIMEOUT` cancel-reason** â€” `RideCancelReason` is exactly `RIDER_CANCELLED | DRIVER_BACKED_OUT | NO_DRIVERS_AVAILABLE | ALL_PASSENGERS_LEFT | MERGED_INTO_ANOTHER_RIDE`.
- **Best-fit (Phase 2e):** proximity-dominant scoring â€” `BEST_FIT = { PROXIMITY_SAME_ZONE: 10, PROXIMITY_ADJACENT: 5, PROXIMITY_TWO_HOP: 1, PROXIMITY_FAR: 0, ROUTE_ALIGNMENT_BONUS: 1 }` ([packages/shared/src/ranking/bestFit.ts](packages/shared/src/ranking/bestFit.ts)). The +1 route-alignment bonus (driver's zone on the pickupâ†’dropoff adjacency path) can only break ties between equally-proximate drivers â€” spacing guarantees it never lets a farther driver outrank a closer one. Ties share `bestFit: true`; â‰¥1 driver is always `bestFit: true` if any eligible drivers exist. This is a driver-side-only scoring concept (`designateBestFit`) â€” it does **not** broadcast anything; wiring the `bestFit` flag into a Socket.io event and the driver UI is deferred to Phase 5.
- **Competitive guarantees (encoded as tests in `packages/shared/src/pricing/pricing.test.ts`):** no shared rider ever pays more than GHS 10 (1000 pesewas, the 1-occupancy rate); a lone-shared driver's car total is â‰¥ 1000; a full (4-seat) car charges 500/rider.
